# VentureVibe - Business Idea Validation Platform (formerly IdeaValidator)

## Project Overview
VentureVibe is a web application that helps entrepreneurs and side hustlers validate their business ideas quickly (within 48 hours) through:
- AI-powered micro-survey generation
- Automated competitor analysis
- Market sizing estimation
- Comprehensive validation reports

## Tech Stack
- **Frontend:** React 18 with Next.js 14 App Router
- **Database:** PostgreSQL with Prisma ORM
- **Authentication:** NextAuth.js v4 with email/password
- **AI:** Abacus AI LLM APIs for content generation and analysis
- **Styling:** Tailwind CSS 3 + custom design tokens
- **Charts:** Recharts for market data visualization

## Key User Flows

### 1. Authentication
- Landing page with signup/login CTAs
- Signup: Email, password, name
- Login: Email, password
- Protected dashboard redirects logged-out users to /auth/login
- Successful auth redirects to /dashboard

### 2. Idea Submission
- Form with: title, description, target audience, problem statement, category
- On submit: Save idea to DB, redirect to dashboard with "pending" status
- No auto-generation - user sees "Ready to validate" CTA on dashboard

### 3. Validation Process
- User clicks "Validate Idea" on dashboard → Navigates to report page
- Report page triggers 3 independent analyses via separate API routes:
  - Survey Generation (API GET /api/validation/survey?ideaId=X) - streams questions
  - Competitor Analysis (API GET /api/validation/competitors?ideaId=X) - streams analysis
  - Market Sizing (API GET /api/validation/market?ideaId=X) - streams market data
- Each uses SSE (Server-Sent Events) with JSON buffering for streaming
- Results saved to ValidationReport table
- Report page shows: survey insights, competitor analysis, market data, overall score, recommendations

### 4. Dashboard View
- Shows all user ideas in grid/list
- Each idea card shows: title, category, status badge, date submitted
- Status: "Pending Validation" → "In Progress" (during analysis) → "Completed"
- Click idea to go to report page

### 5. Validation Report Page
- Displays all 3 analyses (survey, competitors, market sizing)
- Uses recharts for market data visualization
- Shows validation score (0-100) and recommendation
- Lists action items for next steps
- Option to download or share report

## API Routes Architecture

### Authentication
- `POST /api/signup` - User registration (handled by NextAuth setup)
- `POST /api/auth/login` - Handled by NextAuth [...nextauth] callback
- Session management via JWT strategy

### Validation APIs (Streaming with JSON buffering)
- `GET /api/validation/survey?ideaId=X` - Generates survey questions (streams SSE)
- `GET /api/validation/competitors?ideaId=X` - Analyzes competitors (streams SSE)
- `GET /api/validation/market?ideaId=X` - Estimates market size (streams SSE)

### Utility APIs
- `POST /api/ideas` - Submit new idea
- `GET /api/ideas` - Get user's ideas
- `GET /api/ideas/[id]` - Get idea details
- `GET /api/validation/report?ideaId=X` - Fetch existing validation report

## Monetization & Subscription System
- **Stripe Integration** for payment processing
- **Three tiers:** Free (2 validations/month, $0), Pro (15/month, $29), Business (unlimited, $79)
- **Pricing page** at /pricing with Stripe Checkout redirect
- **Usage tracking:** Subscription model tracks validationsUsedThisMonth with monthly auto-reset
- **Gating:** Survey generation route checks limits before first validation of an idea
- **Webhook:** /api/stripe/webhook handles checkout.session.completed, subscription.updated/deleted, invoice.payment_failed
- **Billing portal:** /api/stripe/portal for subscription management
- **Dashboard:** Shows current plan, usage bar, upgrade/manage billing buttons
- **Plans config** separated in lib/plans.ts (safe for client import) vs lib/stripe.ts (server-only Stripe instance)
- Env vars: STRIPE_SECRET_KEY, STRIPE_PUBLISHABLE_KEY, STRIPE_WEBHOOK_SECRET, STRIPE_PRO_PRICE_ID, STRIPE_BUSINESS_PRICE_ID

## Enhanced Features

### Revenue Simulator (Enhanced)
- Scenario Presets: Conservative / Base (AI) / Aggressive one-click toggles derived from AI baseline
- Compare Scenarios: Save up to 3 scenarios, compare MRR over time with line chart + comparison table
- Interactive Sliders: 7 adjustable parameters with real-time 24-month projections

### Market Signals (Enhanced)
- Signal History Timeline: Each Refresh Signals appends a snapshot to marketSignalsHistory (keeps last 12), displayed as a line chart + badge timeline
- Weekly Email Digest: Scheduled task runs every Monday 9 AM UTC, calls /api/cron/market-signals-digest with CRON_SECRET, sends styled HTML digest to all users with active market signals
- Notification: Uses registered notification type NOTIF_ID_WEEKLY_MARKET_SIGNALS_DIGEST

## Important Notes
- JWT strategy with session callbacks to preserve user.id
- Streaming APIs use SSE with JSON-buffering pattern
- All analyses triggered by user action on report page, not auto-fire
- Design favors simplicity over complexity
- No research subtask needed - competitor data comes from LLM analysis
- All validation data persisted in DB for historical tracking
